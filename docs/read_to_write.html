修改权限

Condition

默认来说, 组织的普通成员对 repository 访问权限是只读, 即有浏览代码的权限, 没有修改的权限。 这样的权限管理是比较推荐的, 普通成员可以通过 pull request 来提交自己的代码, 团队所有者可以进行代码检查, 以防止出现不好的代码。

Cause

成员需要修改的权限。

Remedy

组长可以修改。

  1. 选择Settings，选中Collaborators and teams，点击manage。
  2. 选择Member privileges，选中Read，点击Write。
  3. 权限已经修改。
